Edmic E-trading Project Overview:

Asset and Financial management company that deals in trading stocks and shares. Over the years, the organization only attend to inperson customers request (customers do not have options for self-service). The Management Staff have decided to push out a web application for self-service request.

As a business analyst,I designed a product requirements document for this new product

Problem statement:

Lack of self-service request application for trading shares and stock.

Proposed Solution

Edmic will put in place an easy-to-use online self-service platform for daily trading of stock and shares.

Product Vision and goal

To be the leading self- service platform company in trading shares and stock management, enhance Customer Satisfaction and Improve operational efficiency.

Target Audience

Both new and old financial investors.

Product Features

  • User Registration and Secure Login
  • Profile Management
  • Request Submission
  • Request Tracking
  • Notifications
  • Document Upload

Functional Requirements

  • User Registration & Authentication: Allow customers to register online by providing personal details, providing secure password and collecting KYC in line with regulatory requirement.
  • User Login: Allow Registered customers to log in using their username and email address.
  • Profile Management: Allow log in users to view and update their records
  • Account Overview: Displays users investment account, showing current and overall balance, account statement download option.
  • Trading Option: View market data of real time stock and share prices.
  • Place Buy Orders: Users to be able to create buy orders.

Non-Functional Requirements

  • Responsiveness: Page loading time should be in 2-3 seconds
  • Efficiency: The application should utilize server resources very well.
  • Confidentiality:All sensitive user data (personal information, financial must be protected.)
  • Availability: The web application should available to users 99% of the time
  • Compliance: The application must meet all regulatory requirements.
  • Satisfaction:: Users must be okay with the application and there should be room for feedback.

Success Metrics

  • Number of Registered Self-Service Users: Tracks the growth of users actively utilizing the web application.
  • Number of Active Users: The active users who login and interact with the web application should be tracked.
  • Frequency of Logins per User: How often users access the platform.

Release Criteria

  • Functionality Implementation: All the functionalities must be implemented and working very well.
  • Data Migration: Data migration from the previous system must be done successful.

A summarised PRD for the online self-service

This PRD outlines the development of a self-service web application for a stock and share trading company to enhance customer satisfaction and operational efficiency by allowing online request initiation and tracking.
